# 编辑合同解构 PRD

## 需求背景

### 痛点
- **问题现象**：合同解构需要录入多个解构科目（科目编码/名称/税率/金额等），当前无可视化编辑界面
- **发生频率**：高（每合同一次）
- **当前 workaround**：通过Excel录入后批量导入

### 业务目标
- **量化指标**：解构编辑效率提升，减少导入错误率，目标错误率从 5% 降至 1%
- **目标期限**：2026-Q2

### 涉及系统/模块
- **模块名称**：编辑合同解构
- **变更类型**：新增
- **对接接口**：合同解构详情查询接口 / 解构方案保存提交接口

---

## 用户故事

### 故事1
- **角色**：项目经理/财务人员
- **功能**：在编辑页面管理解构明细科目，可添加/删除/修改科目，系统自动计算税额和含税金额并与合同总金额对比
- **收益**：可视化编辑减少出错，金额对比确保解构完整
- **验收条件**：添加科目后自动计算税额和含税金额；价税总计与合同金额对比提示一致性

## 数据流图

### 接口1：获取解构方案（编辑用）
- **请求路径**：`GET /api/contract-demolition/detail`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `demolitionId` - 类型：字符串；必填：是；来源：路由参数；校验：
- **响应字段**：
  - `record` - 类型：对象；描述：合同解构记录
  - `record.contractAmount` - 类型：数字；描述：合同总金额
  - `record.items[]` - 类型：数组；描述：解构明细列表
- **存储位置**：数据库表 contract_demolition / contract_demolition_item
- **错误码**：
  - `401` - `无权限`
  - `404` - `解构记录不存在`
  - `500` - `服务器异常`

### 接口2：保存并提交审批
- **请求路径**：`POST /api/contract-demolition/save`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `demolitionId` - 类型：字符串；必填：是；来源：路由参数；校验：
  - `items[]` - 类型：数组；必填：是；来源：用户编辑的解构明细；校验：至少一项，金额合计≈合同总金额
  - `items[].itemCode` - 类型：字符串；必填：是；来源：用户输入的科目编码；校验：
  - `items[].itemName` - 类型：字符串；必填：是；来源：用户输入的科目名称；校验：
  - `items[].taxRate` - 类型：字符串；必填：是；来源：用户选择的税率；校验：
  - `items[].amount` - 类型：数字；必填：是；来源：用户输入的金额；校验：正数
  - `items[].description` - 类型：字符串；必填：否；来源：用户输入的科目说明；校验：
  - `description` - 类型：字符串；必填：否；来源：解构说明textarea；校验：
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
  - `demolitionId` - 类型：字符串；描述：解构记录ID
- **存储位置**：数据库表 contract_demolition / contract_demolition_item
- **错误码**：
  - `400` - `解构科目金额合计与合同总金额不一致`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 数据刷新点
- **刷新时机**：页面加载时 / 编辑过程中实时计算 / 保存后
- **影响字段**：税额/含税金额/汇总数据

## 验收标准

### 正常流程
- [ ] **操作**：从详情页点击"编辑解构" → **预期**：进入编辑页面，合同基本信息只读显示
- [ ] **操作**：查看现有解构明细 → **预期**：每行显示科目编码/名称/税率/金额等，可编辑
- [ ] **操作**：修改金额 → **预期**：税额和含税金额自动重新计算
- [ ] **操作**：修改税率 → **预期**：税额和含税金额自动重新计算
- [ ] **操作**：点击"添加项目" → **预期**：新增一行空白的解构科目卡片
- [ ] **操作**：点击删除按钮 → **预期**：该行被删除，汇总数据更新
- [ ] **操作**：查看汇总区域 → **预期**：科目数/金额合计/税额合计/价税总计实时显示
- [ ] **操作**：查看金额一致性 → **预期**：与合同总金额对比，一致时显示绿色"✓金额一致"

### 异常流程
- [ ] **操作**：不填写科目编码直接保存 → **预期**：提示"科目编码不能为空"
- [ ] **操作**：价税总计与合同金额差额较大时保存 → **预期**：提示金额不一致
- [ ] **操作**：点击取消 → **预期**：返回详情页，无数据变更
