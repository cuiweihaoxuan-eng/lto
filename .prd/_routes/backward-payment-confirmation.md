# 后向付款确认 PRD

## 需求背景

### 痛点
- **问题现象**：后向付款确认流程需要管理多个合同付款记录、关联列账信息、编辑关联金额，当前无统一管理界面
- **发生频率**：高（每日多次）
- **当前 workaround**：通过Excel记录付款和关联信息，跨系统手动同步

### 业务目标
- **量化指标**：付款确认效率提升 50%，关联错误率降低至 1% 以下
- **目标期限**：2026-Q2

### 涉及系统/模块
- **模块名称**：后向付款确认
- **变更类型**：新增
- **对接接口**：SAP付款数据接口 / 列账信息接口

---

## 用户故事

### 故事1
- **角色**：财务人员/项目经办人
- **功能**：在统一页面管理后向付款确认记录，包括新增付款、关联列账、编辑关联金额、删除记录
- **收益**：付款确认操作集中化，关联关系清晰可追溯，汇总数据实时可见
- **验收条件**：点击新增付款按钮可打开添加SAP付款明细弹窗；点击关联按钮可打开关联列账信息弹窗；展开关联列账记录可编辑本次付款金额

## 数据流图

### 接口1：获取付款确认列表
- **请求路径**：`GET /api/payment/confirmation-list`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `projectCode` - 类型：字符串；必填：否；来源：页面筛选条件；校验：
- **响应字段**：
  - `list[]` - 类型：数组；描述：付款确认单列表
  - `list[].id` - 类型：字符串；描述：记录ID
  - `list[].projectName` - 类型：字符串；描述：项目名称
  - `list[].paymentAmount` - 类型：数字；描述：收付款金额
  - `list[].associatedRecords[]` - 类型：数组；描述：关联列账记录
  - `summary` - 类型：对象；描述：汇总数据
  - `summary.totalAmount` - 类型：数字；描述：项目总金额（万元）
  - `summary.accountPayable` - 类型：数字；描述：应付账款金额（万元）
- **存储位置**：数据库表 payment_confirmation_records
- **错误码**：
  - `401` - `无权限`
  - `500` - `服务器异常`

### 接口2：新增付款确认
- **请求路径**：`POST /api/payment/add-confirmation`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `sapRecordIds[]` - 类型：字符串数组；必填：是；来源：用户勾选的SAP付款明细；校验：至少选一条
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
  - `recordId` - 类型：字符串；描述：新增的付款确认单ID
- **存储位置**：数据库表 payment_confirmation_records
- **错误码**：
  - `400` - `未选择任何记录`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 接口3：关联列账信息
- **请求路径**：`POST /api/payment/associate-accounting`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `paymentRecordId` - 类型：字符串；必填：是；来源：当前付款确认单ID；校验：
  - `accountingRecordIds[]` - 类型：字符串数组；必填：是；来源：用户勾选的列账记录；校验：至少选一条
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
- **存储位置**：数据库表 payment_accounting_association
- **错误码**：
  - `400` - `未选择任何记录`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 接口4：更新关联金额
- **请求路径**：`PUT /api/payment/update-associate-amount`
- **请求方法**：PUT
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `associationId` - 类型：字符串；必填：是；来源：关联记录ID；校验：
  - `currentPaymentAmount` - 类型：数字；必填：是；来源：用户输入的本次付款金额；校验：正数
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
- **存储位置**：数据库表 payment_accounting_association
- **错误码**：
  - `400` - `金额必须为正数`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 接口5：删除关联记录
- **请求路径**：`DELETE /api/payment/association/{id}`
- **请求方法**：DELETE
- **请求头**：Authorization
- **请求参数**：
  - `id` - 类型：字符串；必填：是；来源：关联记录ID；校验：
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
- **存储位置**：数据库表 payment_accounting_association
- **错误码**：
  - `401` - `无权限`
  - `404` - `记录不存在`
  - `500` - `服务器异常`

### 数据刷新点

- **影响字段**：付款确认单列表 / 汇总数据 / 关联列账记录数

## 验收标准

### 正常流程
- [ ] **操作**：进入后向付款确认页面 → **预期**：汇总区域显示项目总金额、应付账款、累计确认付款、剩余未付数据
- [ ] **操作**：点击"新增付款确认" → **预期**：添加SAP付款明细弹窗打开，默认尺寸1600×850
- [ ] **操作**：在弹窗中勾选记录后点击确认 → **预期**：弹窗关闭，主列表刷新，新记录出现在列表中
- [ ] **操作**：点击某行的"关联"按钮 → **预期**：关联列账信息弹窗打开
- [ ] **操作**：在关联弹窗中勾选记录后点击确认 → **预期**：弹窗关闭，该行关联列账记录数更新
- [ ] **操作**：点击"X条"展开关联明细 → **预期**：内嵌明细表格展开显示关联的列账记录
- [ ] **操作**：编辑本次付款金额 → **预期**：已付款金额和剩余未付款金额联动计算

### 异常流程
- [ ] **操作**：不勾选任何SAP记录直接确认 → **预期**：提示"请至少选择一条记录"
- [ ] **操作**：点击删除按钮 → **预期**：弹出确认对话框，确认后删除记录并刷新列表
- [ ] **操作**：输入本次付款金额为负数 → **预期**：前端校验提示金额必须为正数
- [ ] **操作**：点击取消按钮 → **预期**：弹窗关闭，无数据变更
