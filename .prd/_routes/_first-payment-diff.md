# 收款与付款进度差异报表 PRD

## 需求背景

### 痛点
- **问题现象**：项目经理难以直观掌握项目的收款与付款进度是否匹配，收付款进度差异率（净现金流）无法快速获取，需要跨多个财务模块手工比对。
- **发生频率**：高
- **当前 workaround**：财务人员通过 Excel 分别导出收款台账和付款台账，手工计算差异率，数据时效性差。

### 业务目标
- **量化指标**：报表加载时间 < 3s；收付款进度差异率自动计算，准确率 100%
- **目标期限**：2026-Q2 上线

### 涉及系统/模块
- **模块名称**：收款与付款进度差异报表（FirstPaymentDiff）
- **变更类型**：新增
- **对接接口**：`GET /api/reports/first-payment-diff`

## 用户故事

### 故事1
- **角色**：项目经理 / 财务人员
- **功能**：按地市、区县、账期、帐套等多维度筛选，查看各项目的收款进度与付款进度对比情况
- **收益**：一个界面同时展示收款计划、累计实收、付款计划、累计付款、收付款进度差异率、净现金流等关键指标
- **验收条件**：筛选"杭州 + 2026-03 + 股份公司"，表格展示杭州地区项目，净现金流列正值显示蓝色、负值显示红色

### 故事2
- **角色**：财务总监
- **功能**：快速识别净现金流为负数的项目（支出进度远高于收款进度），触发风险预警
- **收益**：通过颜色标记和排序，快速定位高风险项目，优先安排收款跟进
- **验收条件**：收付款进度差异率 > 10% 或净现金流 < 0 的行高亮显示，差异率列支持点击排序

### 故事3
- **角色**：项目经理
- **功能**：按合同维度查看前向合同信息（合同编码/名称/类型/签约日期/签约金额）和后向合同信息（分包采购信息）
- **收益**：在一个报表中同时看到项目的收款方合同和付款方合同，便于匹配分析
- **验收条件**：表格同时包含前向合同信息区（前向合同编号/名称/类型/签约日期）和后向合同信息区（分包合同编号/名称）

## 验收标准

### 正常流程
- [ ] **操作**：页面加载 → **预期**：表格显示当前账期的收款与付款进度差异数据，共21列，四级表头分组显示（项目基本信息/收款/付款/进度差异）
- [ ] **操作**：选择"杭州 + 股份公司 + 2026-03"后点击查询 → **预期**：表格数据按条件过滤

- [ ] **操作**：净现金流 > 0 的行 → **预期**：净现金流列显示蓝色数字

- [ ] **操作**：点击表格中的项目编号 → **预期**：展开该项目详情面板（showDetail=true 生效）

- [ ] **操作**：查询接口返回 401 → **预期**：页面顶部显示"未授权，请重新登录"，跳转登录页

- [ ] **操作**：网络断开时点击查询 → **预期**：显示"网络异常"提示，表格数据保持不变
