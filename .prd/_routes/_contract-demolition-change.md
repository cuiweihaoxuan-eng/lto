# 合同解构变更 PRD

## 需求背景

### 痛点
- **问题现象**：已生效的合同解构方案在执行过程中可能需要调整，当前无变更申请流程
- **发生频率**：中（每月数次）
- **当前 workaround**：通过线下沟通后手动修改数据库记录

### 业务目标
- **量化指标**：变更操作有记录可追溯，变更后自动失效旧方案并使新方案生效
- **目标期限**：2026-Q2

### 涉及系统/模块
- **模块名称**：合同解构变更
- **变更类型**：新增
- **对接接口**：合同解构详情查询接口 / 变更申请提交接口

---

## 用户故事

### 故事1
- **角色**：项目经理/财务人员
- **功能**：在合同解构详情页面点击"申请变更"，进入变更页面，填写变更类型/原因、上传支持文件，系统自动展示变更前后对比，提交后进入审批流程
- **收益**：变更有记录，审批通过后新旧方案自动切换
- **验收条件**：变更前后的解构明细完整对比显示；变更差额自动计算；提交后跳转到详情页

## 数据流图

### 接口1：获取变更前方案
- **请求路径**：`GET /api/contract-demolition/detail`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `demolitionId` - 类型：字符串；必填：是；来源：路由参数；校验：
- **响应字段**：
  - `record` - 类型：对象；描述：当前解构记录
  - `record.currentItems[]` - 类型：数组；描述：当前生效的解构明细
  - `record.newItems[]` - 类型：数组；描述：变更后的解构明细
  - `record.currentTotal` - 类型：数字；描述：变更前总额
  - `record.newTotal` - 类型：数字；描述：变更后总额
  - `record.difference` - 类型：数字；描述：变更差额
- **存储位置**：数据库表 contract_demolition
- **错误码**：
  - `401` - `无权限`
  - `404` - `解构记录不存在`
  - `500` - `服务器异常`

### 接口2：提交变更申请
- **请求路径**：`POST /api/contract-demolition/change`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: multipart/form-data
- **请求参数**：
  - `demolitionId` - 类型：字符串；必填：是；来源：路由参数；校验：
  - `changeType` - 类型：字符串；必填：是；来源：用户选择的变更类型；校验：
  - `changeReason` - 类型：字符串；必填：是；来源：用户输入的变更原因；校验：不为空
  - `attachments[]` - 类型：文件数组；必填：否；来源：用户上传的支持文件；校验：
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
  - `changeId` - 类型：字符串；描述：变更申请ID
- **存储位置**：数据库表 contract_demolition_change
- **错误码**：
  - `400` - `变更原因不能为空`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 数据刷新点
- **刷新时机**：页面加载时 / 提交变更申请后
- **影响字段**：变更前后明细数据 / 汇总数据

## 验收标准

### 正常流程
- [ ] **操作**：从合同解构详情页点击"申请变更" → **预期**：页面进入变更编辑模式，显示当前生效方案（变更前）
- [ ] **操作**：选择变更类型"金额变更"，填写变更原因 → **预期**：变更基本信息区域完成
- [ ] **操作**：查看变更后明细 → **预期**：新增项显示绿色背景，金额调整显示黄色背景
- [ ] **操作**：查看汇总对比 → **预期**：变更前/后总额及差额自动计算显示
- [ ] **操作**：上传支持文件 → **预期**：文件上传区域显示上传进度和文件名
- [ ] **操作**：点击提交变更申请 → **预期**：跳转回详情页，状态变为"变更中"

### 异常流程
- [ ] **操作**：不填写变更原因直接提交 → **预期**：提示"变更原因不能为空"
- [ ] **操作**：点击取消 → **预期**：返回详情页，无数据变更
- [ ] **操作**：上传超过10MB的文件 → **预期**：提示"文件大小不能超过10MB"
