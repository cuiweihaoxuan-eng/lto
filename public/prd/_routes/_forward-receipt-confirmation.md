# 前向收款确认 PRD

## 需求背景

### 痛点
- **问题现象**：收款认领后，需要将认领金额与合同的确收记录进行关联匹配，确认哪些确收已被收款覆盖。当前系统无法直观看到收款与确收的关联关系，用户需要在多个模块间来回切换。
- **发生频率**：高
- **当前 workaround**：财务人员通过 Excel 表格手工匹配收款记录与确收记录，关联关系维护在表格中，容易出错且无法追溯。

### 业务目标
- **量化指标**：收款确认操作耗时 < 30s/笔；关联匹配准确率 > 98%；未收款金额自动计算
- **目标期限**：2026-Q2 上线

### 涉及系统/模块
- **模块名称**：前向收款确认（ForwardReceiptConfirmation）
- **变更类型**：新增
- **对接接口**：`GET /api/receipt/confirm-list`、`POST /api/receipt/associate`

## 用户故事

### 故事1
- **角色**：财务人员
- **功能**：新增一条收款确认记录，填写项目编码、银行流水号、收款认领金额、资金用途（销账/预存款）等信息，确认后将收款与合同的确收记录关联
- **收益**：收款确认全流程在线完成，无需线下 Excel，关联关系实时可查
- **验收条件**：点击"新增收款确认"按钮后弹出 AddReceiptDialog，填写完整后提交，数据保存并刷新列表

### 故事2
- **角色**：财务人员
- **功能**：查看收款确认列表，每条记录显示项目信息、前向合同信息、银行流水号、认领金额、关联确收记录数量
- **收益**：一个界面掌握所有收款确认记录，支持按项目、合同等维度筛选
- **验收条件**：列表显示16列数据（项目编码/名称/前向合同编号/名称/银行流水号/收款认领金额/认领日期/资金用途/认领人/利润中心/成本中心等），关联记录数列固定右侧

### 故事3
- **角色**：财务人员
- **功能**：点击主表行的"关联"按钮，打开关联确收弹窗，将收款记录与一条或多条确收记录进行关联；同时可展开"关联确收记录"查看已关联的明细
- **收益**：收款与确收的关联关系可视化，支持增删改关联记录，剩余未收款金额自动计算
- **验收条件**：点击"关联"按钮后弹出 AssociateReceiptDialog，选择要关联的确收记录后提交，已关联记录在子表格中显示

### 故事4
- **角色**：财务人员 / 项目经理
- **功能**：在页面顶部查看汇总指标区：项目总金额、应收账款金额、累计确认收款金额、剩余未收款金额，以及按合同分列的汇总指标
- **收益**：收款进度一目了然，无需进入详情即可掌握项目整体收款健康度
- **验收条件**：顶部汇总区按项目总览（4个指标）+ 合同1汇总（4个指标）+ 合同2汇总（4个指标）分区显示，金额精确到小数点后两位

## 数据流图

### 接口1：收款确认列表查询
- **请求路径**：`GET /api/receipt/confirm-list`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `projectCode` - 类型：字符串；必填：否；来源：查询条件字段；校验：非空
  - `contractCode` - 类型：字符串；必填：否；来源：查询条件字段；校验：非空
- **响应字段**：
  - `id` - 类型：字符串；描述：记录ID
  - `projectCode` - 类型：字符串；描述：项目编码
  - `claimAmount` - 类型：数字；描述：收款认领金额
  - `associatedRecords` - 类型：数组；描述：关联确收记录列表
- **存储位置**：数据库表 `receipt_confirmation`
- **错误码**：
  - `401` - `未授权，请重新登录`
  - `500` - `服务器异常，请稍后重试`

### 接口2：收款关联确收记录
- **请求路径**：`POST /api/receipt/associate`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `receiptId` - 类型：字符串；必填：是；来源：当前操作的记录ID；校验：非空
  - `confirmRecordId` - 类型：字符串；必填：是；来源：弹窗中选择的确收记录ID；校验：非空
  - `currentReceiptAmount` - 类型：数字；必填：是；来源：弹窗中输入的本次收款金额；校验：>=0
- **响应字段**：
  - `success` - 类型：布尔；描述：操作是否成功
  - `remainingAmount` - 类型：数字；描述：更新后的剩余未收款金额
- **存储位置**：数据库表 `receipt_confirm_association`
- **错误码**：
  - `400` - `收款金额不能大于剩余未收款金额`
  - `401` - `未授权，请重新登录`
  - `500` - `服务器异常，请稍后重试`

### 数据刷新点
- **刷新时机**：页面加载 / 新增保存后 / 关联提交后 / 删除后
- **影响字段**：汇总指标区所有字段、主列表表格、子表格（展开行）

## 验收标准

### 正常流程
- [ ] **操作**：进入页面 → **预期**：顶部汇总区显示12个指标（项目总览4个+合同1汇总4个+合同2汇总4个），主表显示收款确认列表
- [ ] **操作**：点击"新增收款确认"按钮 → **预期**：AddReceiptDialog 弹窗打开，填写后提交保存，列表刷新，新记录出现在列表末尾
- [ ] **操作**：点击主表行的"展开"图标 → **预期**：嵌套子表格展开，显示11列关联确收记录，子表格行背景为浅蓝色
- [ ] **操作**：点击主表行的"关联"按钮 → **预期**：AssociateReceiptDialog 弹窗打开
- [ ] **操作**：点击主表行的"删除"按钮 → **预期**：弹出确认对话框，确认后删除记录，列表刷新，汇总指标更新
- [ ] **操作**：点击子表行的"编辑"按钮 → **预期**：本次收款金额输入框变为可编辑状态

### 异常流程
- [ ] **操作**：不填写必填字段直接提交新增 → **预期**：相应字段下方显示"不能为空"提示
- [ ] **操作**：网络断开时点击提交 → **预期**：显示"网络异常"提示，数据未保存
- [ ] **操作**：新增接口返回 500 → **预期**：弹窗显示错误信息"服务器异常，请稍后重试"，数据未保存
- [ ] **操作**：关联接口返回"收款金额大于剩余未收款" → **预期**：弹窗内显示"收款金额不能大于剩余未收款金额"提示
