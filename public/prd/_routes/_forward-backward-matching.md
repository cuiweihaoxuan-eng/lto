# 前后向匹配 PRD

## 需求背景

### 痛点
- **问题现象**：前后向匹配模块尚在开发中，用户在菜单中可见入口但无法使用实际功能，无法完成前向合同与后向合同的自动匹配分析。
- **发生频率**：中
- **当前 workaround**：用户通过手工方式比对前向合同与后向合同条目，匹配效率低且易出错。

### 业务目标
- **量化指标**：模块上线后，匹配准确率 > 95%；匹配耗时 < 5s/项目
- **目标期限**：2026-Q3 上线

### 涉及系统/模块
- **模块名称**：前后向匹配（ForwardBackwardMatching）
- **变更类型**：新增
- **对接接口**：`POST /api/contract/match`（待实现）

---

## 用户故事

### 故事1
- **角色**：项目经理 / 采购人员
- **功能**：将一个前向合同与一个或多个后向合同进行自动匹配，查看匹配结果（金额/项目/利润/进度是否一致）
- **收益**：自动化前后向合同匹配流程，减少人工核对工作量，避免因匹配错误导致的成本超支
- **验收条件**：输入前向合同编号，点击匹配，系统返回匹配的合同列表及匹配度评分

### 故事2
- **角色**：财务分析人员
- **功能**：导出匹配结果报表，包含匹配金额差异、利润贡献等字段
- **收益**：快速生成供管理层审阅的匹配分析报告
- **验收条件**：点击"导出"按钮后，浏览器下载匹配结果 Excel 文件

## 数据流图

### 接口1：前后向合同匹配（待实现）
- **请求路径**：`POST /api/contract/match`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `forwardContractCode` - 类型：字符串；必填：是；来源：页面输入；校验：非空
  - `matchMode` - 类型：字符串；必填：否；来源：页面选择；校验：枚举值
  - `amountThreshold` - 类型：数字；必填：否；来源：页面输入；校验：>=0
- **响应字段**：
  - `forwardContractCode` - 类型：字符串；描述：前向合同编号
  - `backwardContracts` - 类型：数组；描述：匹配的后向合同列表
  - `matchRate` - 类型：数字；描述：匹配度百分比
  - `matchStatus` - 类型：字符串；描述：匹配状态
- **存储位置**：数据库表 `contract_match_records`（待建）
- **错误码**：
  - `400` - `前向合同编号不能为空`
  - `404` - `未找到对应的合同`
  - `500` - `服务器异常，请稍后重试`

### 数据刷新点
- **刷新时机**：点击"匹配"按钮后 / 导出后
- **影响字段**：匹配结果表格、匹配统计指标

## 验收标准

### 正常流程
- [ ] **操作**：进入页面（开发中状态）→ **预期**：显示"前后向匹配"标题 + AlertCircle 图标 + "该模块正在开发中..."提示文字
- [ ] **操作**：模块开发完成后，输入前向合同编号"HT-2026-001"点击匹配 → **预期**：接口被调用，返回匹配结果表格，包含匹配度/金额差异等字段
- [ ] **操作**：匹配结果中存在不匹配项 → **预期**：不匹配行高亮显示，弹出告警提示
- [ ] **操作**：点击"导出"按钮 → **预期**：浏览器下载 Excel 文件，内容包含所有匹配结果

### 异常流程
- [ ] **操作**：不输入前向合同编号直接点击匹配 → **预期**：字段下方显示"前向合同编号不能为空"
- [ ] **操作**：输入不存在的合同编号 → **预期**：显示"未找到对应的合同"提示
- [ ] **操作**：网络断开时点击匹配 → **预期**：显示"网络异常"提示
- [ ] **操作**：匹配接口返回 500 → **预期**：显示"服务器异常，请稍后重试"
