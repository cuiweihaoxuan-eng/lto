# 专家调用报表 PRD

## 需求背景

### 痛点
- **问题现象**：专家调用过程分散在多个系统，专家资源被调用后是否接单、是否实际参与项目缺乏统一跟踪，项目经理无法量化专家贡献度。
- **发生频率**：高
- **当前 workaround**：管理员通过线下表格记录专家调用情况，数据分散且不实时。

### 业务目标
- **量化指标**：专家贡献度统计准确率 100%；报表加载时间 < 3s
- **目标期限**：2026-Q2 上线

### 涉及系统/模块
- **模块名称**：专家调用报表（ExpertReportPage）
- **变更类型**：新增
- **对接接口**：`GET /api/reports/expert-call-list`、`GET /api/reports/expert-call-stats`

## 用户故事

### 故事1
- **角色**：项目经理
- **功能**：查看专家调用清单，了解哪些专家被哪个项目调用、是否接单、是否实际参与、参与时间
- **收益**：一个界面掌握所有专家调用记录，便于追溯专家资源使用情况
- **验收条件**：切换到"专家调用清单" Tab 后，表格展示所有调用记录，每条记录显示派单人、专家、接单状态、参与状态

### 故事2
- **角色**：专家资源管理员
- **功能**：查看专家统计报表，按专家维度查看派单次数、申请接单次数、接单次数、实际参与次数、累计总贡献度
- **收益**：量化每位专家的贡献度贡献，支持按贡献度排序，识别高价值专家
- **验收条件**：切换到"专家调用统计" Tab 后，表格显示各专家的统计数据，点击列头可升序/降序排序

### 故事3
- **角色**：项目经理
- **功能**：在专家统计报表中，点击某专家的累计总贡献度数值，跳转查看该专家的全部调用明细
- **收益**：快速下钻到专家级别的调用记录详情
- **验收条件**：点击贡献度数字后，当前页面或新窗口展示该专家所有调用记录列表

## 数据流图

### 接口1：专家调用清单查询
- **请求路径**：`GET /api/reports/expert-call-list`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `dispatchTimeStart` - 类型：字符串；必填：否；来源：查询条件字段；校验：YYYY-MM-DD
  - `dispatchTimeEnd` - 类型：字符串；必填：否；来源：查询条件字段；校验：YYYY-MM-DD
  - `expertName` - 类型：字符串；必填：否；来源：查询条件字段；校验：非空
  - `acceptStatus` - 类型：字符串；必填：否；来源：查询条件字段；校验：枚举值
- **响应字段**：
  - `id` - 类型：数字；描述：记录ID
  - `taskName` - 类型：字符串；描述：任务名称
  - `dispatcher` - 类型：字符串；描述：派单人
  - `acceptStatus` - 类型：字符串；描述：接单状态
  - `actuallyJoined` - 类型：字符串；描述：是否实际参与
- **存储位置**：数据库表 `expert_call_list`
- **错误码**：
  - `401` - `未授权，请重新登录`
  - `500` - `服务器异常，请稍后重试`

### 接口2：专家调用统计查询
- **请求路径**：`GET /api/reports/expert-call-stats`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `dispatchTimeStart` - 类型：字符串；必填：否；来源：查询条件字段；校验：YYYY-MM-DD
  - `dispatchTimeEnd` - 类型：字符串；必填：否；来源：查询条件字段；校验：YYYY-MM-DD
  - `expertName` - 类型：字符串；必填：否；来源：查询条件字段；校验：非空
  - `contributionMin` - 类型：数字；必填：否；来源：查询条件字段；校验：>=0
  - `contributionMax` - 类型：数字；必填：否；来源：查询条件字段；校验：>=0
- **响应字段**：
  - `expertName` - 类型：字符串；描述：专家名称
  - `dispatchCount` - 类型：数字；描述：调用派单次数
  - `totalContribution` - 类型：数字；描述：累计总贡献度
- **存储位置**：数据库表 `expert_call_stats`
- **错误码**：
  - `401` - `未授权，请重新登录`
  - `500` - `服务器异常，请稍后重试`

## 验收标准

### 正常流程
- [ ] **操作**：进入页面，默认 Tab 为"专家调用清单" → **预期**：表格显示专家调用清单数据，16列
- [ ] **操作**：点击"专家调用统计" Tab → **预期**：内容切换为统计报表，8列表格，支持列头排序
- [ ] **操作**：在"专家调用清单" Tab 填写"专家名称=李华"，点击查询 → **预期**：表格仅显示李华的调用记录
- [ ] **操作**：点击"展开更多条件" → **预期**：显示"商机编码"查询条件
- [ ] **操作**：点击"重置"按钮 → **预期**：所有查询条件恢复默认值，表格刷新
- [ ] **操作**：点击"累计总贡献度"列头 → **预期**：表格按贡献度降序排列，图标变为降序
- [ ] **操作**：再次点击"累计总贡献度"列头 → **预期**：切换为升序排列

### 异常流程
- [ ] **操作**：查询接口返回 401 → **预期**：页面顶部显示"未授权，请重新登录"
- [ ] **操作**：查询接口返回 500 → **预期**：表格区域显示"服务器异常，请稍后重试"
- [ ] **操作**：网络断开时点击查询 → **预期**：显示"网络异常"提示
- [ ] **操作**：派单时间起 > 止 → **预期**：字段下方显示"开始日期不能大于结束日期"
- [ ] **操作**：输入非数字的贡献度范围 → **预期**：字段下方显示"请输入数字"
