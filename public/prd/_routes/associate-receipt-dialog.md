# 关联确收信息 PRD

## 需求背景

### 痛点
- **问题现象**：收款确认后需要将收款记录与确收信息关联，当前无界面化操作方式
- **发生频率**：中（每周数次）
- **当前 workaround**：财务人员通过财务系统查询确收凭证，手动记录关联关系

### 业务目标
- **量化指标**：关联操作时间从 10 分钟/次降至 3 分钟/次
- **目标期限**：2026-Q2

### 涉及系统/模块
- **模块名称**：关联确收信息
- **变更类型**：新增
- **对接接口**：确收信息查询接口（待对接）

---

## 用户故事

### 故事1
- **角色**：财务人员/项目经办人
- **功能**：通过弹窗筛选并勾选确收信息记录，建立与收款确认单的关联关系
- **收益**：减少手动录入的工作量，关联关系可追溯
- **验收条件**：勾选确收记录后点击确认，关联关系被建立，收款确认单页面刷新显示关联信息

## 数据流图

### 接口1：查询确收信息
- **请求路径**：`GET /api/revenue/confirmed-items`
- **请求方法**：GET
- **请求头**：Authorization
- **请求参数**：
  - `projectCode` - 类型：字符串；必填：否；来源：当前收款确认单关联的项目编码；校验：
- **响应字段**：
  - `records[]` - 类型：数组；描述：确收信息列表
  - `records[].id` - 类型：字符串；描述：记录ID
  - `records[].projectName` - 类型：字符串；描述：项目名称
  - `records[].confirmedIncomeWithTax` - 类型：数字；描述：已确认收入金额（含税）
  - `records[].associatedReceiptAmount` - 类型：数字；描述：已关联收款金额
  - `records[].accountingSubject` - 类型：字符串；描述：会计科目
- **存储位置**：财务系统（数据来源）/ LTO本地缓存
- **错误码**：
  - `401` - `无权限`
  - `500` - `确收接口调用失败`

### 接口2：确认关联
- **请求路径**：`POST /api/receipt/associate-confirmed`
- **请求方法**：POST
- **请求头**：Authorization / Content-Type: application/json
- **请求参数**：
  - `receiptRecordId` - 类型：字符串；必填：是；来源：当前收款确认单ID；校验：
  - `confirmedItemIds[]` - 类型：字符串数组；必填：是；来源：用户勾选的确收记录ID列表；校验：至少选一条
- **响应字段**：
  - `success` - 类型：布尔；描述：是否成功
  - `associatedCount` - 类型：数字；描述：关联的记录数量
- **存储位置**：数据库表 receipt_confirmed_association
- **错误码**：
  - `400` - `未选择任何记录`
  - `401` - `无权限`
  - `500` - `服务器异常`

### 数据刷新点
- **刷新时机**：点击确认后
- **影响字段**：列表数据 / 收款确认单列表

## 验收标准

### 正常流程
- [ ] **操作**：点击某收款确认单的"关联确收"按钮 → **预期**：弹窗打开，默认尺寸1500×750，列表显示可关联的确收信息
- [ ] **操作**：点击全选复选框 → **预期**：所有行被勾选
- [ ] **操作**：勾选部分记录后点击确认 → **预期**：弹窗关闭，列表页刷新，该收款确认单已关联选中的确收记录

### 异常流程
- [ ] **操作**：不勾选任何记录直接点击确认 → **预期**：提示"请至少选择一条记录"
- [ ] **操作**：点击取消 → **预期**：弹窗关闭，无数据变更
- [ ] **操作**：拖动弹窗标题栏移动位置 → **预期**：弹窗跟随鼠标移动
- [ ] **操作**：拖动弹窗边缘调整大小 → **预期**：弹窗尺寸变化（不小于最小尺寸）
